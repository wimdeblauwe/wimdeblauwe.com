Migrating away from Thymeleaf Layout Dialect

2026-02-25 #thymeleaf #spring-boot

Interested in learning more about Thymeleaf? Check out my book Taming Thymeleaf. The book combines all of my Thymeleaf knowledge into an easy-to-follow, step-by-step guide.

This blog post explains what the Thymeleaf Layout Dialect is, why you might want to stop using it, and how to migrate to a pure Thymeleaf solution using th:replace.

What is Thymeleaf Layout Dialect?

The Thymeleaf Layout Dialect is a third-party extension for Thymeleaf that adds a decorator-based layout system to your templates. The idea is that you define a single layout template (the "decorator") that contains the common HTML structure of your pages: the <head>, a navigation bar, a footer, and so on.

Each content page then declares which layout it wants to be decorated with:

src/main/resources/templates/index.html 
<html xmlns:layout="https://www.ultraq.net.nz/thymeleaf/layout"
      layout:decorate="~{layout/main}">
<body>
  <div layout:fragment="content">
    <h1>Welcome</h1>
  </div>
</body>
</html>

Why move away from it?

The layout dialect has served the community well for over a decade. But there are good reasons to reconsider it for new projects:

Every dependency you add is a dependency you must understand, upgrade, and one day migrate away from.

Pure Thymeleaf can express the same layouts using fragment expressions with parameters. Since Thymeleaf 3, th:replace accepts fragment expressions that pass markup as arguments, which covers the decorator use case without any third-party dependency.

Migrating to th:replace

Instead of decorating pages, you invert the relationship: the page calls the layout and hands it the content as a fragment parameter.

src/main/resources/templates/index.html 
<html xmlns:th="http://www.thymeleaf.org"
      th:replace="~{layout/main :: layout(~{::title}, ~{::main})}">
<head>
  <title>Home</title>
</head>
<body>
  <main>
    <h1>Welcome</h1>
  </main>
</body>
</html>

The result is one less dependency, faster startup, and templates that any Thymeleaf user can read without knowing the dialect's conventions.
